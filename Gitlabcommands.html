Git easy now...!

Using GIT from command line in a GNU/Linux OS.

Let us try to clone a remote project and do some experiments using GIT commands in a GNU/Linux OS.

First off, you are here because you already have a gitlab account, atleast I assumee so. If not, don't delay. Sign up. I tried to make this as engaging as possible, as you should feel the author is explaining you rather reading an article

You have your GITLAB account setup now. Now, let's go to terminal, CTRL+ALT+T will take you to terminal/konsole.

Just want to type some commands. Ho! Wait. You need to have some basic requirements to be there beforehand. But what are they? How would you know, right! I help you dive in. Just see if the below conditions are satisfied. Else, click on them.

  1. 1. Working Linux Environment.
  2. 2. GIT installed.
  3. 3. Configuring your GIT with SSH keys.

Now, we start from very basic stuff..

1. Search for the GIT version by
git --version

It should display *git version 2.17.1*

If GIT is not installed in your local, don't worry, here it is. *https://about.gitlab.com/install/*

2. It’s time to configure in your local system.

git config --global user.name "akhil"
git config --global user.email "akhil@swecha.net"

Note : --global tells your Gitlab to use this information always for anything you do on your Computer.

Good. Now the configuration is done. But how do I check whether it’s done or not? Just try this and see what you get

git config --global --list

3. Cloning part is on the way. It’s very simple.

git clone code.swecha.org/akhil/Project-name .

The ‘.’ in the above command says to clone the project into the current directory

Tadaa! The project is cloned.
Note : Try git remote -v and git branch -b, they gives the information about the remote repository and banches respectively.

4. Everything is fine, does the git knows the folders that we have just cloned is a repository.
No, because it’s just a folder in my Local. That’s where initialising part comes.

git init

This command instructs git to start to track that folder/directory as a repository.

5. Commit changes :
create an empty file through terminal.
touch test and write some code or text in the file.
To check the changes made/current status :
git status
To add all the files :
git add -A
In case, if you want to add only one or specific file
git add *file name*
Let’s commit these changes :
git commit -m “first commit”

6. We are almost done. The final steps are to push the code into the original repo.
git push origin master
